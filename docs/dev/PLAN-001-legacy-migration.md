# PLAN-001 旧库迁移与审阅隔离

## 背景

当前主仓库为 `/Users/mac/pai-studio/rlearn_dev.py`，已同步到远端 `origin/main` 的最新提交 `af39257`。

旧库位于 `/Users/mac/turing/repository/rlearn_dev.py`，当前本地分支落后远端 2 个提交，并存在本地未提交改动、未跟踪调试脚本、外部参考包和训练产物。

## 目标

以当前主仓库为准，迁移旧库中可确认有价值的代码修复；暂时无法确定是否需要的旧库材料复制到当前主仓库的独立审阅目录，使旧库目录后续可以安全删除。

## 实施范围

1. 迁移旧库本地可确认的代码修复：
   - `PPOAgent` 训练日志中的 `approx_kl` 从张量转为 Python 数值，避免后续均值和日志处理混入 tensor。
   - 连续动作空间网络在 action space 上下界包含无穷值时禁用 action scaling，避免无穷边界参与缩放。
2. 保留当前主仓库已存在的远端最新结构：
   - 不回退 `rlearn_dev/methods/ppo/draft/agent.py` 的重命名。
   - 不直接采用旧库中的测试实验改动。
3. 建立本地审阅区：
   - 保存旧库本地 tracked diff。
   - 保存旧库本地变更文件快照。
   - 保存旧库未跟踪 debug 脚本。
   - 保存旧库外部参考包 `packages/cleanrl`，排除其 `.git` 元数据。
   - 保存旧库被 ignore 的训练产物，排除缓存、`__pycache__` 和 egg-info。
4. 解除当前主仓库对旧库路径的依赖：
   - 移除当前仓库内指向旧库的未跟踪符号链接 `rlearn_dev.py`。
   - 将审阅区加入 `.gitignore`，避免大型审阅材料误提交。

## 验证

1. 运行 `git diff --check` 检查新增空白问题。
2. 运行聚焦单元测试，覆盖连续动作空间无限边界禁用缩放逻辑。
3. 检查当前仓库不再包含指向旧库路径的符号链接。
4. 生成结果文档，记录已迁移、已归档和未处理事项。
